# РУКОВОДСТВО ПО СБОРУ ИНФОРМАЦИИ ИЗ СКРИНШОТОВ

## ПРОЦЕСС РАБОТЫ

1. **Вы называете инструмент** (например: "Cursor")
2. **Вы присылаете скриншот(ы)** настроек этого инструмента
3. **Я анализирую скриншот** и извлекаю информацию
4. **Я добавляю информацию** в базу знаний в структурированном виде

## ПРИОРИТЕТНЫЙ СПИСОК ИНСТРУМЕНТОВ ДЛЯ ДОКУМЕНТИРОВАНИЯ

### ВЫСОКИЙ ПРИОРИТЕТ (КРИТИЧНЫЕ ИНСТРУМЕНТЫ)

1. **Cursor** - настройки IDE, расширения, AI модели
2. **Continue** - настройки моделей, API keys, конфигурация
3. **Warp** - настройки терминала, workflows, tasks
4. **Ollama** - установленные модели, настройки сервера
5. **Claude Desktop** - MCP серверы, настройки подключений
6. **n8n** - credentials, webhooks, настройки окружения
7. **Dify** - API keys, модели, настройки RAG
8. **Bitrix24** - REST API настройки, webhooks, интеграции

### СРЕДНИЙ ПРИОРИТЕТ (ВАЖНЫЕ ИНСТРУМЕНТЫ)

9. **Obsidian** - настройки vault, плагины, hotkeys
10. **GitHub** - токены, SSH keys, настройки репозиториев
11. **Telegram** - настройки ботов, API keys
12. **Google Drive** - настройки синхронизации, пути
13. **blink.new** - настройки интеграции с n8n

### НИЗКИЙ ПРИОРИТЕТ (ДОПОЛНИТЕЛЬНЫЕ)

14. **iOS Shortcuts** - список и настройки автоматизаций
15. **Yandex Disk** - настройки rclone
16. **Comss.one DNS** - настройки DoT
17. **WireGuard** - конфигурация VPN (если доступна)

## ЧТО ВАЖНО ИЗ СКРИНШОТОВ НАСТРОЕК

### ДЛЯ КАЖДОГО ИНСТРУМЕНТА НУЖНО

1. **Основные настройки:**
   - Версия/версия приложения
   - Пути к файлам/директориям
   - Порт/адрес (если применимо)
   - Режим работы (production/dev)

2. **API и интеграции:**
   - API keys (только названия переменных, не значения!)
   - Endpoints/URLs
   - Webhooks
   - OAuth настройки

3. **Модели и AI:**
   - Используемые модели
   - Провайдеры (OpenRouter, Ollama, etc.)
   - Настройки температуры, токенов
   - Embeddings модели

4. **Расширения и плагины:**
   - Список установленных расширений
   - Настройки расширений
   - Hotkeys

5. **Сети и безопасность:**
   - VPN настройки
   - Proxy настройки
   - SSL/TLS сертификаты
   - Firewall правила

6. **Хранилище и синхронизация:**
   - Пути к данным
   - Настройки синхронизации
   - Бэкапы

## ШАБЛОН ДЛЯ КАЖДОГО ИНСТРУМЕНТА

При добавлении информации о новом инструменте я буду использовать следующую структуру:

```markdown
### [Название инструмента]
- **Категория:** [из существующей классификации]
- **Версия:** [если видна]
- **Статус:** активно/частично/в работе
- **Критичность:** высокая/средняя/низкая

**Настройки:**
- [Параметр 1]: [значение или описание]
- [Параметр 2]: [значение или описание]

**API/Интеграции:**
- [Тип интеграции]: [описание, без секретов]

**Модели/AI (если применимо):**
- [Модель 1]: [назначение]
- [Модель 2]: [назначение]

**Расширения/Плагины:**
- [Название]: [назначение]

**Связи:**
- [Связь с другим компонентом]: [тип связи]

**Проблемы/Замечания:**
- [если есть]
```

## БЕЗОПАСНОСТЬ

⚠️ **ВАЖНО:** 
- Не присылайте скриншоты с реальными API keys, паролями, токенами
- Если на скриншоте есть секреты - замаскируйте их перед отправкой
- Я буду документировать только названия переменных окружения, не их значения

## ПОРЯДОК РАБОТЫ

Начнем с инструментов высокого приоритета. Когда будете готовы, просто напишите:

**"Начинаем с [название инструмента]"** и пришлите скриншот(ы).

Я буду задавать уточняющие вопросы, если что-то будет неясно на скриншоте.

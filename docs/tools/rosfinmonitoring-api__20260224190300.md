# ROSFINMONITORING API (115-ФЗ)

## 1) ОПИСАНИЕ

**Федеральная служба по финансовому мониторингу (Росфинмониторинг)** - публикует
перечень организаций и физических лиц, связанных с терроризмом и экстремистской
деятельностью.

**Правовая база:** Федеральный закон от 07.08.2001 № 115-ФЗ «О противодействии
легализации (отмыванию) доходов, полученных преступным путём, и финансированию
терроризма».

**Обязанность:** Финансовые организации и организации, совершающие операции с
денежными средствами, обязаны проверять контрагентов перед заключением договоров
и блокировать операции с лицами из перечня.

---

## 2) ТЕХНИЧЕСКИЕ ХАРАКТЕРИСТИКИ

### 2.1 Источник данных

**Официальный сайт:** https://www.fedsfm.ru

**XML-перечень:** https://www.fedsfm.ru/opendata/7708234633-perechen

**Страница с перечнями:** https://www.fedsfm.ru/documents/terrorists-catalogues-portal-act

### 2.2 Формат данных

**Формат:** XML

**Структура:**
- Физлица: ФИО, ИНН, дата рождения, описание
- Юрлица: наименование, ИНН, ОГРН, описание

**Размер:** ~5-10 МБ (зависит от количества записей)

**Обновление:** нерегулярное (раз в несколько дней/недель)

### 2.3 Доступ

**Авторизация:** Не требуется (открытые данные)

**Ограничения:** Сайт блокирует программные запросы (403 Forbidden)

**Решение:** Скачивание через браузер или headless browser

## 3) РЕАЛИЗАЦИЯ В КОМПАНИИ

### 3.1 Скрипт проверки

**Путь:** `~/Documents/ai_agents/scripts/rosfinmonitoring-check.py`

**Возможности:**
- Проверка по ФИО
- Проверка по ИНН
- Проверка по наименованию организации
- Локальный кеш XML-перечня
- Логирование всех проверок

**Зависимости:**
```bash
pip install requests lxml
```

**Пример использования:**
```bash
python3 scripts/rosfinmonitoring-check.py \
  --action check \
  --name "Иванов Иван Иванович" \
  --format json
```

### 3.2 Интеграция с Bitrix24

**Workflow n8n:**
1. Trigger: создание/изменение контакта в B24
2. Extract: ФИО, ИНН из контакта
3. Check: запуск rosfinmonitoring-check.py
4. Update B24: сохранение результата в поля UF_RFM_*
5. Alert: уведомление при обнаружении совпадения

**Поля B24:**
- `UF_RFM_CHECK`: OK | BLOCKED | PENDING
- `UF_RFM_CHECK_DATE`: дата последней проверки
- `UF_RFM_MATCHES`: детали совпадений (JSON)

## 4) COMPLIANCE-ТРЕБОВАНИЯ

### 4.1 Обязательные проверки

**Когда проверять:**
- При первичном контакте (до заключения договора)
- При изменении реквизитов контрагента
- Повторно раз в год для действующих договоров

### 4.2 Действия при обнаружении

**Если лицо/организация в перечне:**
1. БЛОКИРОВАТЬ операцию (не заключать договор)
2. Уведомить Data & Compliance Officer + QA Risk Officer + CEO
3. НЕ сообщать контрагенту причину отказа (115-ФЗ, ст. 7)
4. Зафиксировать в логах и B24

### 4.3 Хранение данных

**Логи проверок:** минимум 3 года (требование 115-ФЗ)

**Путь логов:** `~/Documents/ai_agents/logs/rosfinmonitoring.log`

**Формат логов:** JSON-структура с timestamp, query, result

---

## 5) ИЗВЕСТНЫЕ ОГРАНИЧЕНИЯ

1. **Блокировка программных запросов:** Сайт возвращает 403 Forbidden при попытке
   загрузки через curl/requests. Решение: ручное скачивание или headless browser.

2. **Нерегулярное обновление:** Росфинмониторинг обновляет перечень без чёткого
   расписания. Локальная копия может быть устаревшей.

3. **Неточность сравнения:** ФИО могут совпадать у разных людей. Обязательна
   ручная проверка деталей при совпадении.

4. **Изменение формата XML:** Структура XML может измениться без предупреждения.

## 6) АЛЬТЕРНАТИВЫ

### 6.1 Ручная проверка

**URL:** https://www.fedsfm.ru/documents/terrorists-catalogues-portal-act

**Способ:** Поиск по ФИО/ИНН в браузере

**Недостатки:** Медленно, нет автоматизации, нет логирования

### 6.2 Агрегаторы данных

**api-parser.ru** - включает проверку по перечню Росфинмониторинга в составе
комплексной проверки контрагента (990 руб/мес).

**SpectrumData** - аналогично включает проверку в составе пакета проверок
(~10 руб/запрос).

**Преимущества:** Автоматическая загрузка, актуальные данные, комплексная проверка.

**Недостатки:** Платные, зависимость от внешнего сервиса.

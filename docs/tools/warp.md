## Использование в проектах

### Интеграция с проектами базы знаний

Warp используется как операторская панель для работы с инфраструктурой в следующих проектах:

#### 1. vps/
- **Файл правил:** `vps/WARP.md`
- **Назначение:** управление VPS инфраструктурой через Warp Agent
- **Приоритет источников:**
  1. Warp-слой (env vars, workflows, prompts, rules)
  2. Документация в репозитории (`docs/context.md`, `docs/docker-map.md`, `docs/changes.md`)
  3. `docs/vars.md` (справочник переменных)
- **Стартовый протокол:** `vps/docs/start_here.md`
- **Основные команды:**
  - `make status` - проверка статуса
  - `make ps` - список контейнеров
  - `make logs` - просмотр логов

#### 2. n8n/
- **Файл правил:** `n8n/WARP.md`
- **Назначение:** работа с n8n workflows через Warp Agent

#### 3. n8n_workflow/
- **Файл правил:** `n8n_workflow/WARP.md`
- **Назначение:** управление leadgen workflows

#### 4. Beget2/
- **Файл правил:** `Beget2/WARP.md`
- **Назначение:** управление инфраструктурой на Beget

### Принципы работы с Warp Agent

1. **Источник истины:**
   - Локальный репозиторий (`<repo-root>`) - для кода и конфигурации
   - Документация в `docs/*.md` - для знаний и состояния
   - Warp env vars / workflows - для операционных переменных

2. **Приоритет источников:**
   - Warp-слой (ускоряет работу агента)
   - Документация репозитория (source of truth)
   - `docs/vars.md` (справочник для сверки)

3. **Безопасность:**
   - Секреты только в `secrets/*.env` (не в репозитории)
   - Разрушительные действия требуют подтверждения
   - Пароли не логируются и не копируются

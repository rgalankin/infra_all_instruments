# 2GIS API Platform

## Основная информация

- **Категория:** API - Geo & Maps (геоданные, навигация, поиск организаций)
- **Статус:** активно (демо-ключ получен)
- **Критичность:** средняя (вспомогательный инструмент для CRM-обогащения)
- **Назначение:** поиск организаций, геокодирование, маршруты, зоны доступности по России и СНГ
- **Платформа:** cross-platform (REST API, JS SDK, iOS/Android/Flutter SDK)
- **Документация:** https://docs.2gis.com/en/api
- **Личный кабинет:** https://platform.2gis.ru/

## Подключение

- **Auth:** API key (передается в query parameter `key`)
- **Credentials:** `.secrets/geo_api_keys.json` (демо-ключ)
- **Base URLs:**
  - Search API: `https://catalog.api.2gis.com/3.0/`
  - Navigation API: `https://routing.api.2gis.com/`
  - Map Tiles: `https://tile.api.2gis.com/`
  - Static API: `https://static.api.2gis.com/`

## Демо-ключ (текущий)

| Параметр | Значение |
|----------|----------|
| Тип | Demo |
| Лимит | 1 000 запросов на КАЖДЫЙ сервис |
| Срок | 1 месяц с момента активации |
| Map Tiles | 500 000 запросов |
| Static API | 30 000 запросов |
| Ключ хранится | `.secrets/geo_api_keys.json` |

**Ограничения демо-ключа:**
- Заблокированы платные методы: `/3.0/items/byphone`, `/3.0/items/byitin`, `/3.0/items/bysite`, `/3.0/items/byfias`, `/3.0/items/bytradelicense`
- Заблокированы платные поля: `contact_groups`, `itin`, `trade_license`, `fias_code`, `employees_org_count`
- 1 000 запросов на каждый сервис (не на все суммарно)

## Примеры запросов

### Поиск организаций (Places API)
```bash
# Поиск кафе рядом с координатами (Москва)
curl "https://catalog.api.2gis.com/3.0/items?q=cafe&location=37.630866,55.752256&key=YOUR_KEY"

# Поиск банков в радиусе 1 км
curl "https://catalog.api.2gis.com/3.0/items?q=bank&location=37.630866,55.752256&radius=1000&key=YOUR_KEY"

# Поиск по категории (рубрике)
curl "https://catalog.api.2gis.com/3.0/items?rubric_id=RUBRIC_ID&location=37.630866,55.752256&key=YOUR_KEY"
```

### Геокодирование (Geocoder API)
```bash
# Прямое: адрес -> координаты
curl "https://catalog.api.2gis.com/3.0/items/geocode?q=Москва,Тверская+17&fields=items.point&key=YOUR_KEY"

# Обратное: координаты -> адрес
curl "https://catalog.api.2gis.com/3.0/items/geocode?lat=55.752256&lon=37.630866&fields=items.full_name&key=YOUR_KEY"
```

### Зоны доступности (Isochrone API)
```bash
curl --request POST \
  --url 'https://routing.api.2gis.com/isochrone/2.0.0?key=YOUR_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "start": {"lat": 55.752256, "lon": 37.630866},
    "durations": [600, 1200, 1800],
    "reverse": false,
    "transport": "driving"
  }'
```

### Distance Matrix API
```bash
curl --request POST \
  --url 'https://routing.api.2gis.com/distance_matrix?key=YOUR_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "sources": [{"lat": 55.75, "lon": 37.63}],
    "targets": [{"lat": 55.76, "lon": 37.64}, {"lat": 55.74, "lon": 37.62}],
    "transport": "driving"
  }'
```

Требуют отдельной покупки доступа у отдела продаж 2GIS:

|--------------|-----------|----------------------|
| `/3.0/items/byphone` | Поиск организаций по телефону | **ВЫСОКАЯ** (обратный поиск по городскому номеру) |
| `/3.0/items/byitin` | Поиск организаций по ИНН | **ВЫСОКАЯ** (верификация контрагентов) |

| `/3.0/items/byfias` | Поиск по коду ФИАС | Средняя |
| `items.contact_groups` | Детальная контактная информация | Высокая |

## Сценарии использования для Кредо-Сервис / Банкрот-Сервис

### Приоритет P1 (высокий, есть бизнес-ценность)

#### 1. Верификация адресов клиентов (Geocoder API)
- **Задача:** При CRM-миграции (38K контактов из 4 источников) нормализовать и верифицировать адреса
- **Как:** Прямое геокодирование адресов -> координаты -> обратное геокодирование -> нормализованный адрес
- **Ценность:** Очистка базы от некорректных/дублирующих адресов, стандартизация формата
- **Ограничение:** 1 000 запросов на демо. Для 38K контактов нужен платный тариф или DaData (10K/день бесплатно)
- **Рекомендация:** Для массовой верификации использовать DaData (бесплатный лимит 10K/день). 2GIS Geocoder оставить для точечных задач.

#### 2. Обогащение CRM данных (Places API)
- **Задача:** Найти организации рядом с адресами клиентов; определить, какие банки-партнёры есть в городе клиента
- **Как:** Поиск по категории "банк" + координатам города клиента
- **Ценность:** Автоматический подбор банка-партнёра по геолокации клиента (128 банков-партнёров)
- **Пример:** Клиент из Новосибирска -> найти филиалы банков-партнёров в Новосибирске -> предложить ближайший

#### 3. Поиск организаций по ИНН (byitin, ПЛАТНЫЙ)
- **Задача:** Верификация работодателей клиентов (кредитный брокеридж)
- **Как:** `/3.0/items/byitin?itin=<ИНН>&key=KEY`
- **Ценность:** Получение полной карточки организации из 2GIS (адрес, часы работы, рубрики)
- **Ограничение:** ЗАБЛОКИРОВАНО на демо-ключе
- **Альтернатива:** DaData findById (бесплатно, 10K/день) - лучший вариант

### Приоритет P2 (средний, полезно но не критично)

#### 4. Геоаналитика: распределение клиентов (Places API + Isochrone API)
- **Задача:** Визуализировать, где географически сконцентрированы клиенты; определить зоны покрытия офисов
- **Как:** Геокодировать адреса клиентов -> кластеризация -> Isochrone API для зон доступности от офисов
- **Ценность:** Понимание, где нужны новые офисы / партнёры; оптимизация рекламы по гео
- **Пример:** Построить изохрону 30/60 мин от офиса Банкрот-Сервис -> какой % клиентов попадает в зону доступности

#### 5. Маршрутизация выездного обслуживания (TSP API + Directions API)
- **Задача:** Если есть выездные консультации (кредитные менеджеры / юристы)
- **Как:** TSP API для оптимизации маршрута курьера по 5-10 точкам в день
- **Ценность:** Экономия времени на выезды
- **Применимость:** Только если компания реализует выездную модель обслуживания

#### 6. Distance Matrix для логистики документов
- **Задача:** Расчёт расстояний между клиентом, офисом, судом, МФЦ
- **Как:** Distance Matrix API для матрицы расстояний/времени
- **Ценность:** Автоматический подбор ближайшего суда/МФЦ для клиента по банкротству

### Приоритет P3 (низкий, на перспективу)

#### 7. Карты для сайта (MapGL JS API + Map Tiles)
- **Задача:** Интерактивная карта офисов/партнёров на bankrotserv.ru и credoserv.ru
- **Как:** MapGL JS API для встраивания карты с маркерами офисов
- **Ценность:** Визуальная навигация для клиентов

#### 8. Мониторинг банков-партнёров (Places API)
- **Задача:** Отслеживать актуальность данных о банках-партнёрах (адреса, часы работы, статус)
- **Как:** Периодический запрос Places API по списку банков

#### 9. Suggest API для форм на сайте
- **Задача:** Автодополнение адреса при заполнении формы клиентом
- **Как:** Suggest API интегрировать в формы на сайте
- **Альтернатива:** DaData Suggestions (стандарт рынка для РФ)

## Итоговая оценка

| Критерий | Оценка |
|----------|--------|
| Релевантность для бизнеса | **Средняя** |
| Качество данных по РФ | **Высокое** (лучший справочник организаций) |
| Бесплатный лимит | **Низкий** (только демо 1K req) |
| Уникальные возможности vs альтернативы | **Средние** (DaData закрывает большинство кейсов бесплатно) |
| Приоритет подключения | **P2** (после DaData, ФССП, Яндекс Метрика) |

### Когда 2GIS лучше альтернатив:
1. **Детальные данные об организациях** (часы работы, рубрики, рейтинг, фото) - богаче чем DaData
2. **Навигация и маршруты по РФ** (TSP, Isochrone, Distance Matrix) - лучше Яндекс Карт по API-возможностям
3. **Данные внутри зданий** (этажи, планы этажей) - уникально
4. **Покрытие СНГ** (Казахстан, Узбекистан, Беларусь) - если будет экспансия

### Когда DaData лучше:
1. Массовая верификация адресов (10K/день бесплатно vs 1K демо)
2. Поиск по ИНН/телефону (бесплатно vs платные методы 2GIS)
3. Стандартизация ФИО, ИНН, паспортных данных (2GIS этого не умеет)
4. MCP-сервер уже есть (dadata-mcp-server)

## Рекомендация

**Для текущей CRM-миграции (38K контактов):**
- Геокодирование и верификация адресов -> **DaData** (бесплатно, 10K/день, лучше для РФ)
- Поиск по телефону -> **DaData** (byphone заблокирован на демо 2GIS)
- Поиск по ИНН -> **DaData findById** (бесплатно)

**Где использовать 2GIS:**
1. Поиск банков-партнёров в городе клиента (Places API, в рамках 1K демо запросов)
2. Зоны доступности от офисов (Isochrone API, визуализация для маркетинга)
3. Карты на сайте (MapGL JS API, если не хотим Google Maps / Яндекс Карты)
4. Маршруты для выездного обслуживания (TSP/Directions, если запустим выездную модель)

**Вывод:** 2GIS - хороший инструмент с уникальными навигационными API, но для текущих задач CRM-миграции DaData покрывает потребности лучше и дешевле. 2GIS стоит сохранить как вспомогательный инструмент для геоаналитики и навигации. Когда демо-ключ закончится, оценить целесообразность покупки на основе реального использования.

## Риски

- **risk_tier:** low
- **data_allowed:** public, business (адреса, координаты, публичные данные организаций)
- **data_forbidden:** хранение персональных данных клиентов на стороне 2GIS
- **known_failure_modes:**
  1. Демо-ключ истечёт через 1 месяц (нет бесплатного тарифа)
  2. Платные методы (byphone, byitin) заблокированы на демо
  3. Лимит 1 000 запросов на сервис - мало для массовых операций
  4. Покрытие: хорошее по городам РФ, слабее по сельской местности

# ИНФРАСТРУКТУРА IT

Единая карта всей IT инфраструктуры, инструментов, сервисов и их взаимосвязей.

## Назначение

Этот проект содержит:
- **Инвентарь инфраструктуры** - серверы, сети, VPN, DNS
- **Документацию инструментов** - настройки, конфигурации, интеграции
- **Карту взаимосвязей** - как компоненты связаны между собой
- **MCP серверы** - документация всех Model Context Protocol серверов
- **Рекомендации** - как эффективно использовать и развивать инфраструктуру

## Основные файлы

### `ВСЯ МОЯ ИНФРАСТРУКТУРА (для LLM).md`

Основной файл с полным инвентарем инфраструктуры:
- Инфраструктура (серверы, сети, VPN, DNS)
- Программы и приложения (Desktop, Mobile, Browser-based)
- Сервисы и SaaS
- AI и автоматизации
- Knowledge-система
- Интеграции и связи

### `docs/tools/`

Детальная документация каждого инструмента:
- **cursor.md** - Cursor IDE (MCP серверы, расширения)
- **warp.md** - Warp терминал (MCP серверы, интеграции)
- **claude-desktop.md** - Claude Desktop (MCP серверы)
- **n8n.md** - n8n (workflows, credentials, интеграции)
- **paddleocr.md** - PaddleOCR (Docker контейнер для OCR)
- **tesseract-ocr.md** - Tesseract OCR (локальный OCR инструмент)

### `docs/mcp-servers.md`

Документация всех MCP (Model Context Protocol) серверов:
- Cursor (10 серверов)
- Warp (5 серверов)
- Claude Desktop (3 сервера)
- Сравнение и статистика

## Использование

### Для AI агентов

AI агенты могут использовать этот проект для:
- Понимания всей инфраструктуры
- Поиска подходящих инструментов для задач
- Анализа зависимостей между компонентами
- Предложений по улучшению и расширению

### Для разработчиков

Разработчики могут использовать для:
- Понимания архитектуры
- Планирования интеграций
- Документирования новых инструментов
- Стратегического развития инфраструктуры

## Процесс обновления

### Добавление нового инструмента

1. Собрать информацию (скриншоты, настройки)
2. Создать документацию в `docs/tools/[название].md`
3. Обновить основной файл `ВСЯ МОЯ ИНФРАСТРУКТУРА (для LLM).md`
4. Обновить связи в `docs/mcp-servers.md` (если есть MCP серверы)

### Обновление существующего инструмента

1. Обновить соответствующий файл в `docs/tools/`
2. Обновить основной файл при необходимости
3. Зафиксировать изменения

## Связи с другими проектами

## Безопасность

⚠️ **ВАЖНО:**
- В документации **НЕ хранятся** реальные секреты (API keys, пароли, токены)
- Указываются только **названия переменных окружения**, не их значения
- Секреты хранятся только в соответствующих системах (n8n UI, .env файлы и т.д.)

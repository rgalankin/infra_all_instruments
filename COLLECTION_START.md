# Начало сбора информации из скриншотов

## Как мы будем работать

1. **Вы называете инструмент** из списка ниже
2. **Вы присылаете скриншот(ы)** настроек этого инструмента
3. **Я анализирую** скриншот и задаю уточняющие вопросы (если нужно)
4. **Я добавляю информацию** в базу знаний

## С чего начать?

Рекомендую начать с инструментов **высокого приоритета**, так как они наиболее критичны для работы:

### Приоритет 1: Критичные инструменты

1. **Cursor** - основной IDE
2. **Continue** - AI слой в IDE
3. **Warp** - терминал
4. **Ollama** - локальные LLM модели
5. **Claude Desktop** - MCP клиент
6. **n8n** - автоматизация (веб-интерфейс)
7. **Dify** - AI платформа (веб-интерфейс)
8. **Bitrix24** - CRM (веб-интерфейс)

### Приоритет 2: Важные инструменты

9. **Obsidian** - база знаний
10. **GitHub** - управление кодом
11. **Telegram** - коммуникации и боты
12. **Google Drive** - синхронизация
13. **blink.new** - веб-сайт

### Приоритет 3: Дополнительные

14. **iOS Shortcuts** - мобильные автоматизации
15. **Yandex Disk** - хранилище
16. **Comss.one DNS** - DNS настройки
17. **WireGuard** - VPN (если доступны настройки)

## Что нужно на скриншотах?

Для каждого инструмента важно увидеть:

- **Основные настройки** (версия, пути, порты)
- **API/Интеграции** (endpoints, webhooks - **без реальных ключей!**)
- **Модели AI** (если применимо)
- **Расширения/Плагины** (список установленных)
- **Сети/Безопасность** (VPN, proxy настройки)

## Безопасность

⚠️ **ВАЖНО:**
- **Не присылайте** скриншоты с реальными API keys, паролями, токенами
- **Замаскируйте** секреты перед отправкой (закрасьте или замените на `***`)
- Я буду документировать только **названия переменных окружения**, не их значения

## Готовы начать?

Просто напишите:

**"Начинаем с [название инструмента]"**

и пришлите скриншот(ы) настроек.

Например:
- "Начинаем с Cursor" + скриншот Settings
- "Начинаем с Continue" + скриншот конфигурации моделей
- "Начинаем с n8n" + скриншот credentials и webhooks

Я буду задавать уточняющие вопросы, если что-то будет неясно на скриншоте.

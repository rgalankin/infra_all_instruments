# CLAUDE.MD

Этот файл содержит инструкции для Claude Code (claude.ai/code) при работе с данным репозиторием.

## ЧТО ЭТО ЗА РЕПОЗИТОРИЙ

Это **репозиторий документации IT-инфраструктуры** (база знаний), а не программный проект. Содержит структурированный инвентарь всех инструментов, сервисов, серверов, AI-систем, интеграций и их взаимосвязей.

Владелец: Роман Галанкин. Основная машина: iMac Pro 2017 (macOS).

## СТРУКТУРА РЕПОЗИТОРИЯ

- `ВСЯ МОЯ ИНФРАСТРУКТУРА (для LLM)__20260201200225.md` — **основной файл**: полный Tooling Registry (v1.2) со всей инфраструктурой, софтом, SaaS, AI и интеграциями
- `docs/tools/` — детальная документация по каждому инструменту (cursor__20260201200225.md, warp__20260201200225.md, n8n__20260201200225.md, claude-desktop__20260201200225.md, paddleocr__20260201200225.md, tesseract-ocr__20260201200225.md, neural-networks__20260201232828.md)
- `docs/tools/template.md` — шаблон для документирования новых инструментов (использовать при добавлении)
- `docs/mcp-servers__20260201200225.md` — все MCP серверы: Cursor (10), Warp (5), Claude Desktop (3)
- `docs/screenshot-collection-guide__20260201200225.md` — руководство по сбору информации из скриншотов
- `COLLECTION_START__20260201200225.md` — инструкция для начала процесса сбора информации
- `scripts/mcp-sync.mjs` — Node.js скрипт синхронизации MCP конфигов между инструментами
- `.continue/mcpServers/new-mcp-server.yaml` — конфигурация MCP серверов для Continue IDE

## КЛЮЧЕВЫЕ ПРАВИЛА

- **Никаких секретов в документации**: документируются только *названия* переменных окружения, не значения. Секреты хранятся в `.env` файлах и UI инструментов (n8n и т.д.)
- **Source of truth**: локальный диск + Git для кода/инфры. Сервер — среда исполнения, конфиги приходят из репозитория
- **Markdown — стандарт** для всей документации
- **Документация инструментов по шаблону** из `docs/tools/template.md` — использовать при добавлении новых инструментов

## ДОБАВЛЕНИЕ/ОБНОВЛЕНИЕ ДОКУМЕНТАЦИИ ИНСТРУМЕНТА

1. Создать/обновить файл инструмента в `docs/tools/[название].md` по шаблону
2. Обновить `ВСЯ МОЯ ИНФРАСТРУКТУРА (для LLM)__20260201200225.md`
3. Если у инструмента есть MCP серверы — обновить `docs/mcp-servers__20260201200225.md`
4. Обновить перекрёстные ссылки между документами

## СКРИПТЫ

```bash
# Синхронизация MCP конфигов между Cursor, Claude Desktop, VS Code, OpenCode, Codex, Claude Code
node scripts/mcp-sync.mjs              # синхронизация (добавить недостающие серверы)
node scripts/mcp-sync.mjs --dry-run    # предпросмотр без записи
node scripts/mcp-sync.mjs --force      # перезаписать существующие серверы каноничной версией
node scripts/mcp-sync.mjs --status     # показать текущее состояние всех конфигов
node scripts/mcp-sync.mjs --install    # установить launchd агент (каждые 6ч)
node scripts/mcp-sync.mjs --uninstall  # удалить launchd агент
```

Приоритет слияния MCP конфигов: Cursor > Claude Code > VS Code > OpenCode > Codex > Claude Desktop. Бэкапы сохраняются в `~/.config/mcp-sync/backups/`.
